# PLIP AI 리포트 기능 작업사항

> **작업일**: 2026-04-17
> **기능**: CODI API(KT 사내 AI)를 활용한 AI 분석 리포트
> **대상**: ALP/MOP/LLP 데이터 조회 화면에서 현재 보고 있는 차트 데이터를 기반으로 AI가 한글 분석 리포트를 생성

---

## 1. 요구사항

지자체 고객 요청으로 AI 기반 분석 리포트 기능 추가. 기존 데이터 조회 화면(ALP/MOP/LLP)의 `DataHeader`에 "AI 리포트" 버튼을 추가하여, 현재 차트 데이터를 CODI API(KT 사내 Dify 기반 LLM)로 전송하면 스트리밍 방식으로 한글 분석 리포트를 생성한다.

## 2. 아키텍처

```
[FE]
 ├─ AIReportButton.tsx  (버튼 → 모달 오픈)
 └─ AIReportModal.tsx   (SSE 스트리밍 수신 + 렌더링)
        │
        ▼ POST /plip/api/ai/report
[FE BFF]
 └─ src/app/api/ai/report/route.ts  (iron-session 토큰 추출 → BE 프록시)
        │
        ▼ POST /ai/report (Authorization: Bearer)
[BE]
 ├─ router/ai.ts                    (/report 라우트 정의, verifyToken)
 ├─ controller/ai/aiController.ts   (요청 파싱, 프롬프트 빌드, SSE 헤더 세팅)
 └─ service/ai/aiReport.ts          (CODI API 스트리밍 프록시)
        │
        ▼ POST /chat-messages (response_mode: streaming)
[CODI API]
  https://api.codi.kt.co.kr/v1
```

## 4. 주요 구현 포인트

### 4.1. BE - CODI API 프록시 (스트리밍)
- `http`/`https` 모듈 직접 사용 (Node core, streaming 네이티브)
- `rejectUnauthorized: false` — 사내 인증서 대응
- CODI에서 받은 청크를 그대로 `res.write()`로 FE에 전달 (SSE 패스스루)
- 에러 발생 시 `data: { event: "error", message }\n\n` 형식으로 전송

### 4.2. BE - 프롬프트 빌드 (`buildReportPrompt`)
- 리포트 제목, 지역명, 기간, 데이터 유형 메타 정보를 상단에 기재
- 차트 데이터(JSON)를 본문에 삽입
- 출력 형식 지시: 개요 / 주요 지표 / 변화 추이 / 특이사항 / 정책 제언
- **할루시네이션 방지**: "데이터에 있는 수치만 사용하고, 존재하지 않는 수치를 만들어내지 마세요"

### 4.3. FE - 스트리밍 수신
- `fetch` + `ReadableStream.getReader()` + `TextDecoder`로 SSE 파싱
- 라인 버퍼링 처리(마지막 불완전 라인은 다음 chunk와 합침)
- `event: "message" | "agent_message"`의 `answer` 필드만 누적
- `event: "error"` 시 에러 상태 세팅
- `AbortController`로 모달 닫을 때 스트림 취소

### 4.4. FE - 모달 UX
- 로딩 중 스피너, 스트리밍 중 커서 표시 (`animate-pulse`)
- 완료 후 "복사"/"재생성" 버튼 노출
- 자동 스크롤(`scrollTop = scrollHeight`)
- `BaseModal` `width="w-2/3"` 사용

## 5. 작업 중 발견/수정한 결함

작업 중 세션이 튕긴 뒤 이어서 진행하면서 다음 결함을 수정:

### 5.1. `aiReport.ts` — HTTPS 옵션 오타
```diff
- rejectAuthorized: false,
+ rejectUnauthorized: false,
```
Node의 TLS 옵션 이름은 `rejectUnauthorized`. 이전 오타 상태로는 사내 인증서 검증이 여전히 켜져 있어 요청 실패 가능성.

### 5.2. `AIReportModal.tsx` — `buttons` props 타입 불일치
`BaseButton` 인터페이스는 `title`/`color: "outlined"`를 요구하지만, 모달에서 `text`/`color: "border"`로 전달하고 있어 타입 에러:
```diff
- { text: "복사", color: "border" as const, ... }
+ { title: "복사", color: "outlined" as const, ... }
```
또한 spread + ternary 구조를 직접 ternary로 단순화.

### 5.3. `AIReportModal.tsx` — `ChartData` union 타입 안전 접근
`ChartData = BaseChartData | MergedChartData` union이기 때문에 `chart.regionName`, `chart.indicate` 직접 접근 시 타입 에러. `any` 캐스트로 안전하게 우회하고, `data.options` 역시 유니온 타입이므로 `any` 캐스트 + optional 접근 처리.

## 6. 환경변수

### BE (`env` / `env.dev`)
```
CODI_API_URL=https://api.codi.kt.co.kr/v1
CODI_API_KEY=app-xxxx
```

### FE (`.env.local` 등)
기존 `NEXT_PUBLIC_EXPRESS_API_URL`, `NEXT_PUBLIC_BASE_PATH` 사용 — 추가 변수 없음.

## 7. 검증 방법

1. BE/FE 로컬 기동
   ```bash
   cd be-alp-main && npm run dev   # port 8080
   cd fe-alp-main && npm run dev   # port 3000
   ```
2. BDIP 로그인 후 `/plip/alp`, `/plip/mop`, `/plip/llp` 중 아무 데이터 화면 진입
3. 차트가 렌더링된 후 `DataHeader` 우상단의 AI 리포트 아이콘(`report.svg`) 클릭
4. 모달에서 스트리밍 리포트가 점진적으로 렌더링되는지 확인
5. "재생성" / "복사" 버튼 동작 확인
6. 모달 닫을 때 진행 중이던 스트림이 취소되는지 확인
