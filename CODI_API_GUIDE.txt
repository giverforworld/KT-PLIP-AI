대화형 앱 API 가이드
API를 사용하여 최종 사용자 인터페이스를 직접 개발할 수 있습니다.
현재 사용하는 앱 분류는 채팅앱으로, 채팅앱의 경우 세션유지 기능을 지원합니다. ( 챗봇, 고객 상담 등에 적합합니다. )
세션 안에서 발생하는 채팅은 답변을 위한 Context 정보로 사용됩니다.

API URL
https://api.codi.kt.co.kr/v1

Header
인증을 위해 제공되는 API Key를 HTTP의 Authorization Header에 넣어주세요.

ex) Authorization: Bearer YOUR_API_KEY

요청을 보내는 클라이언트를 식별하기 위해 User-Agent Header를 꼭 포함해주세요.

ex) User-Agent: YOUR_APP_NAME/1.0

POST
endpoint: /chat-messages


Request 설명
채팅 메시지를 전송합니다.

Body
query (string): 사용자의 질문
inputs (object): 앱에서 사전 정의한 변수의 key-value 값을 넣습니다. 아무것도 없으면 기본값으로 를 넣습니다.
response_mode (string): streaming( 스트리밍 모드입니다. 타이핑 하는것과 같이 연속적으로 응답이 발생합니다.) <br/>blocking : 블럭킹 모드입니다. 한번에 완성된 응답이 발생합니다 ( 채팅앱에서는 지원하지 않습니다. )
user (string): 사용자 이름 ( 사용자를 인식할 수 있는 Unique한 값이 되어야 합니다 )
converstation_id (string): 대화 아이디, 값을 넣으면 대화를 이어서 하고, 아니면 새로운 대화를 만듭니다.
files (array[object]): 이미지, 이미지를 설명하는 텍스트로 이루어진 파일 리스트 입니다. 사용하는 모델이 vision 기능을 지원해야 작동합니다.
type (string) : 파일타입, 현재는 image 만 가능합니다.
transfer_method (string) : 전송방식, remote_url 또는 local_file 을 넣습니다.
url (string) : remote_url을 선택하였을 경우 url을 넣습니다.
upload_file_id (string) : 파일 업로드 api를 사용한다음 응답받은 id 값을 넣습니다.
auto_generate_name (bool): 대화 제목을 자동으로 생성할지 여부를 결정합니다. 기본값은 false 입니다.
Example
curl


curl -X POST 'https://api.codi.kt.co.kr/v1/chat-messages' \
--header 'Authorization: Bearer {API_KEY}' \
--header 'Content-Type: application/json' \
--data-raw '{
    "inputs": {},
    "query": "[삭제X] PLIP_AI에 대해 알려줘",
    "response_mode": "streaming",
    "conversation_id": "",
    "user": "test1234",
    "files": [
        {
        "type": "image",
        "transfer_method": "remote_url",
        "url": "https://api.codi.kt.co.kr/v1/logo/logo.png"
        }
    ]
}'
Response 설명
streaming 모드의 Response 내용

Body
응답 메시지 청크단위 스트리밍으로 리턴합니다.
Content-Type은 text/event-stream 입니다.
각각의 청크들은 data: 로 시작되고, \n\n 으로 구분됩니다.

event : 메시지 이벤트 타입: 아래 내용은 이벤트 타입에 따른 output 설명 입니다

event : message, agent_message ( 대화형 앱 에서만 작동 ): LLM에 의해 생성된 메시지 이벤트

event (string) : message or agent_message
task_id (string) : 작업 아이디, API 중지를 위해 사용될 수 있습니다.
message_id (string) : 메시지 아이디
conversation_id (string) : 대화 ID
answer (string) : LLM이 응답한 텍스트 일부
created_at (int) : 생성일시
event : message_file: Tools에 의해 새로운 파일이 생성되었을때 발생하는 이벤트

event (string) : message_file
id (string) : 파일 아이디
type (string) : 파일 타입 ( image )
belongs_to (string)
url (string) : 파일 URL
conversation_id (string) : 대화 아이디
event : message_end: 메시지 수신이 완전히 종료되었을때 발생하는 이벤트

event (string) : message_end
task_id (string) : 작업 아이디
message_id (string) : 메시지 아이디
conversation_id (string) : 대화 아이디
metadata (object) : 메타데이터
usage(Usage) : 모델 사용 정보
retriever_resources(array[RetrieverResources]) : 인용, 기여 리스트
event : message_replace: moderation api가 활성화 되어있을 경우 필터링이 필요한 내용을 교체해주는 이벤트

event (string) : message_replace
task_id (string) : 작업 아이디
message_id (string) : 메시지 아이디
conversation_id (string) : 대화 아이디
answer (string) : LLM이 응답한 텍스트 일부
created_at (int) : 생성일시
event : ping: 연결 유지를 위한 ping 이벤트

event : error: 에러 이벤트

event (string) : error
task_id (string) : 작업 아이디
message_id (string) : 메시지 아이디
status (int) : http 상태 코드
code (string) : error 코드
message (string) : error 메시지

에러
에러 종류 및 메시지

Errors
404, Conversation does not exists - 대화가 존재하지 않음
400, invalid_param, abnormal parameter input - 비정상적인 파라미터 입력
400, app_unavailable, App configuration unavailable - 앱 설정을 사용할 수 없음
400, provider_not_initialize, no available model credential configuration - 사용 가능한 모델 자격 증명 구성 없음
400, provider_quota_exceeded, model invocation quota insufficient - 모델 호출 할당량 부족
400, model_currently_not_support, current model unavailable - 현재 모델을 사용할 수 없음
400, completion_request_error, text generation failed - 텍스트 생성 실패
400, file_too_large, File size exceeds maximum limit - 파일 크기가 최대 제한을 초과
400, unsupported_file_type, File type not supported - 지원되지 않는 파일 유형
413, file_upload_failed, File upload failed - 파일 업로드 실패
500, internal server error - 내부 서버 오류
POST
endpoint: /files/upload


Request 설명
채팅 메시지에서 이미지 입력을 위한 파일을 업로드합니다.

Body (multipart / form-data)
file (File): 업로드할 파일 ( 지원 형식: png, jpg, jpeg, webp, gif )
user (string): 사용자 이름 ( 사용자를 인식할 수 있는 Unique한 값이 되어야 합니다 )
Example
curl


curl -X POST 'https://api.codi.kt.co.kr/v1/files/upload' \
--header 'Authorization: Bearer {API_KEY}' \
--form 'file=@"path/to/image.jpg"' \
--form 'user="test1234"'
Response 설명
파일 업로드 Response 내용

Body

name (string): 파일 이름
size (int): 파일 크기 (bytes)
extension (string): 파일 확장자
mime_type (string): MIME 타입
created_by (string): 파일 업로드한 사용자
created_at (int): 생성일시

Request 설명
현재 사용자의 대화 리스트를 조회합니다. 기본적으로 가장 최근의 20건을 조회합니다.

pinned (bool, optional): 즐겨찾기(pinned) 지정된 대화만 조회. true/false
Example
curl


curl -X GET 'https://api.codi.kt.co.kr/v1/conversations?user=test1234&last_id=&limit=20' \
--header 'Authorization: Bearer {API_KEY}' \
